## 👨‍🏫 1. ระบบครูประจำวิชา (Subject Teacher System)
จากโครงสร้างฐานข้อมูลปัจจุบัน สามารถขยายผลเพื่อรองรับระบบครูประจำวิชาได้ดังนี้:

### การปรับปรุง Database
- [ ] **ตาราง `schedule`**: เปลี่ยนคอลัมน์ `teacher_name` (Text) เป็น `teacher_id` (UUID) ที่อ้างอิงไปยัง `profiles.id` เพื่อระบุตัวตนครูที่สอนจริงๆ
- [ ] **ตารางรายวิชา**: พิจารณาสร้างตาราง `subjects` เพื่อเก็บ รหัสวิชา, ชื่อวิชา และคำอธิบาย เพื่อความเป็นมาตรฐาน
- [ ] **ลิงก์ข้อมูล**: ผูก `attendance_logs.subject` เข้ากับ ID ของวิชาแทนการใช้ข้อความ

### การปรับปรุง Logic/UI
- [ ] **Teacher Dashboard**: ปรับให้ครูเห็นตารางสอนของตัวเอง และรายการนักเรียนที่เช็คชื่อเฉพาะในคาบที่ตนเองสอน
- [ ] **RLS Policy**: เพิ่มความปลอดภัยโดยให้ครูที่มี `teacher_id` ตรงกับตารางสอนเท่านั้นที่มีสิทธิ์กด Verify ในคาบนั้นๆ

### การสร้างและจัดการครูประจำวิชา
- [ ] **Admin Teacher Management**: เพิ่มหน้าจอให้ Admin สามารถสร้างบัญชีครู และระบุได้ทันทีว่าครูคนนี้รับผิดชอบวิชาอะไรบ้าง (Subject Assignment)
- [ ] **Role-based Invite**: ระบบเชิญครูเข้าใช้งานผ่าน Email พร้อมกำหนด Role เป็น Teacher ล่วงหน้า
- [ ] **Teacher Profile**: เพิ่มส่วนข้อมูลของครู เช่น ความเชี่ยวชาญ, ตารางสอนส่วนตัว และประวัติการตรวจสอบการเข้าเรียนของนักเรียนในวิชาที่สอน

## ⚙️ 3. การจัดการระบบ (System Architecture)
- [ ] **Centralized Logic**: ย้าย Logic ที่ใช้ซ้ำๆ ไปไว้ใน `/src/lib/` (เหมือนที่ทำ `ui.js` และ `supabaseClient.js`) เพื่อให้ง่ายต่อการดูแล
- [ ] **State Management**: หากโปรเจกต์ใหญ่ขึ้น อาจพิจารณาใช้ระบบจัดการ State เพื่อให้ข้อมูลระหว่างหน้า Sync กันได้เร็วขึ้น
- [ ] **Advanced Analytics**: หน้า Admin Dashboard ควรมีกราฟสรุปสถิติการเข้าเรียนรายเดือน/รายเทอม (เช่น โดยใช้ Chart.js)

## 📱 4. ฟีเจอร์เสริม (Extra Features)
- [ ] **Notification (LINE OA Integration)**: 
  - เนื่องจาก LINE Notify ปิดตัวลง ต้องใช้ **LINE Messaging API** ผ่าน LINE Official Account (OA)
  - **ระบบยืนยันตัวตนผู้ปกครอง (Parent-Student Linking)**:
    1. **Unique Student Key**: ระบบเจนรหัส "Parent Key" (เช่น PIN 6 หลัก) ให้กับนักเรียนแต่ละคนในตาราง `profiles`
    2. **LINE LIFF App**: เมื่อผู้ปกครองแอด LINE OA ให้กดเมนู "ลงทะเบียนรับแจ้งเตือน" ซึ่งจะเปิดหน้าเว็บเล็กๆ (LIFF)
    3. **Linking Process**: ผู้ปกครองกรอก **รหัสนักเรียน** + **รหัส Parent Key** -> ระบบจะบันทึก `line_user_id` ของผู้ปกครองลงในฐานข้อมูลที่ผูกกับนักเรียนคนนั้น
    4. **Push Notification**: เมื่อนักเรียนเช็คชื่อ ระบบจะใช้ Supabase Edge Function ส่งข้อความหา `line_user_id` ที่ผูกไว้ทันที
- [ ] **QR Code Check-in**: ให้ครูเจน QR Code ในคาบเรียน แล้วให้นักเรียนสแกนเพื่อเช็คชื่อแทนการกดเลือกเอง
- [ ] **Export Data**: ระบบส่งออกข้อมูลการเข้าเรียนเป็นไฟล์ Excel หรือ PDF สำหรับทำรายงานสรุปท้ายเดือน

## 🔐 5. ระบบความปลอดภัยและการยืนยันตัวตน (Security & Auth)
ไอเดียที่สอดคล้องกับฟีเจอร์ของ Supabase Auth เพื่อเพิ่มความสะดวกและปลอดภัย:

### ระบบจัดการบัญชี (Account Management)
- [ ] **Forgot Password Flow**:
  - ใช้ `supabase.auth.resetPasswordForEmail()` เพื่อส่งลิงก์รีเซ็ตรหัสผ่านไปยังอีเมล
  - สร้างหน้า `reset-password.html` เพื่อรับ Token และให้ผู้ใช้กรอกรหัสผ่านใหม่
- [ ] **Change Email Logic**:
  - ใช้ `supabase.auth.updateUser({ email: newEmail })` 
  - **Note**: ระบบของ Supabase จะส่งอีเมลยืนยันไปยังทั้ง "อีเมลเก่า" และ "อีเมลใหม่" เพื่อความปลอดภัยสูงสุด (สามารถตั้งค่าใน Dashboard ได้)

### ระบบเชิญผู้ใช้ (Invitation System)
- [ ] **Admin Invite**:
  - ใช้ `supabase.auth.admin.inviteUserByEmail()` สำหรับ Admin ในการส่งคำเชิญให้ครูหรือนักเรียน
  - ผู้รับจะได้อีเมลพร้อมลิงก์ที่คลิกแล้วจะเข้ามาตั้งรหัสผ่านเองครั้งแรก (ลดภาระ Admin ในการตั้งรหัสผ่านชั่วคราว)
- [ ] **Restricted Registration**:
  - ปิดระบบสมัครสมาชิกทั่วไป (Disable Public Sign-up) บน Supabase Dashboard
  - กำหนดให้การเพิ่มผู้ใช้ใหม่ทำได้เฉพาะการ **"Invite via Email"** หรือ **"Admin Create User"** เท่านั้น เพื่อควบคุมคุณภาพสมาชิกในระบบ
- [ ] **Custom Email Templates**: ปรับแต่งข้อความและดีไซน์ใน Email Templates บน Supabase Dashboard ให้เป็นสไตล์ **Neubrutalism** (เช่น ใส่ปุ่มที่มีเส้นขอบหนา) เพื่อให้เข้ากับแบรนด์ของเว็บ

### การปรับปรุง UI/UX หน้า Auth
- [ ] **Show Password Toggle**: เพิ่มปุ่มไอคอน (รูปตา) ในช่องรหัสผ่านหน้า Login/Register เพื่อให้ผู้ใช้สามารถกดดูรหัสที่พิมพ์ได้ ลดความผิดพลาดในการตั้งรหัสผ่าน

## 📊 6. ระบบ Import/Export ข้อมูล (Data Management)
การจัดการข้อมูลจำนวนมากผ่านไฟล์ Excel เพื่อความรวดเร็ว:

### ตารางสอน (Schedule)
- [ ] **Export Schedule**: เพิ่มปุ่มดาวน์โหลดตารางสอนปัจจุบันเป็นไฟล์ Excel (ใช้ Library เช่น `xlsx`)
- [ ] **Import Schedule**: ระบบอัปโหลดไฟล์ Excel เพื่ออัปเดตตารางสอนทีเดียวทั้งเทอม (แทนการกดเพิ่มทีละวิชา)

### การลงทะเบียนครูแบบกลุ่ม (Bulk Teacher Registration)
- [ ] **Bulk Create/Invite via Excel**: 
  - **Option 1 (Invite)**: ส่งคำเชิญให้ครูตั้งรหัสผ่านเอง (ปลอดภัยที่สุด)
  - **Option 2 (Direct Create)**: Admin เป็นคนตั้งรหัสผ่านให้เลย (เช่น ตั้งเป็น 123456 ให้ทุกคนก่อน)
  - **ทำได้จริงหรือไม่?**: **"ทำได้แน่นอนครับ"** แต่มีเงื่อนไขสำคัญ:
    1. **ใช้ Admin API**: ต้องใช้คำสั่ง `supabase.auth.admin.createUser()` พร้อมระบุ `password` และตั้ง `email_confirm: true` เพื่อให้ใช้งานได้ทันทีโดยไม่ต้องรอยืนยันเมล
    2. **Security Warning**: การตั้งรหัสผ่านให้เลย **"ห้ามทำผ่านโค้ดหน้าเว็บ (Frontend) ตรงๆ"** เพราะต้องใช้ `service_role` key ซึ่งเป็นกุญแจสำคัญที่ห้ามหลุดไปที่ Browser
    3. **Edge Functions**: ต้องทำผ่าน Supabase Edge Functions เท่านั้น เพื่อความปลอดภัย Admin ส่งข้อมูล Excel ไปที่ Function แล้วให้ Function เป็นคนสร้าง User ให้ครับ

## 📅 7. ระบบลอจิกการเข้าเรียนขั้นสูง (Advanced Attendance Logic)
การเพิ่มความฉลาดให้กับระบบเช็คชื่อเพื่อความแม่นยำและยืดหยุ่น:

### ประเภทการลา (Conditional Leave Options)
- [ ] **Dynamic Sub-options**: เมื่อนักเรียนเลือกสถานะ **ลากิจ, ลาป่วย, หรือ กิจกรรม** ให้แสดงตัวเลือกเพิ่มเติม:
  - ลาทั้งวัน (Full Day)
  - ครึ่งวันเช้า (Morning)
  - ครึ่งวันบ่าย (Afternoon)
  - ลาเฉพาะคาบ (Specific Period)
- [ ] **Mandatory Reason**: สำหรับการลาทั้ง 3 ประเภทนี้ **"บังคับ"** ให้กรอกเหตุผลในช่อง Note (ห้ามว่าง)
- [ ] **No Time Limit for Leave**: การลาเหล่านี้สามารถกดย้อนหลังหรือล่วงหน้าได้ ไม่จำกัดเวลาและคาบเรียน

### การมาเรียนปกติ (Present Status)
- [ ] **Smart Subject Locking**: เมื่อนักเรียนเลือกสถานะ **"มาเรียน"**:
  - ระบบจะตรวจสอบตารางสอน (`schedule`) ตามวันและเวลาปัจจุบัน (Real-time)
  - **Auto-lock**: ล็อกวิชาเรียนให้โดยอัตโนมัติตามคาบเรียนที่ตรงกับเวลานั้น (นักเรียนไม่ต้องเลือกวิชาเอง)
  - **Time-limit**: ตั้งเวลาจำกัดการเช็คชื่อ (เช่น เช็คได้ไม่เกิน 15 นาทีหลังจากเริ่มคาบ) เพื่อป้องกันการเช็คชื่อนอกเวลาเรียน

## 🛑 8. ระบบจัดการการขาดเรียน (Absence Management)
ไอเดียการจัดการนักเรียนที่ไม่ได้เช็คชื่อ เพื่อให้ข้อมูลครบถ้วน 100%:

### ระบบเช็คขาดอัตโนมัติ (Auto-Absent Cron Job)
- [ ] **End-of-Period Check**: เมื่อจบคาบเรียน (เช่น หลังเริ่มคาบ 20 นาที) ระบบจะรันคำสั่งตรวจสอบอัตโนมัติ
- [ ] **Auto-Mark**: เปรียบเทียบรายชื่อนักเรียนในห้อง กับคนที่เช็คชื่อเข้ามาแล้ว ใครที่ยังไม่มีข้อมูลในคาบนั้น ระบบจะบันทึกสถานะเป็น **"ขาดเรียน (Absent)"** ให้โดยอัตโนมัติ
- [ ] **Notification to Parents**: ทันทีที่ระบบเช็คขาด ให้ส่งข้อความแจ้งเตือนเข้า LINE OA ของผู้ปกครองทันที เพื่อให้ทราบว่าบุตรหลานไม่ได้เข้าเรียนในคาบนั้น

### ระบบ Dashboard สำหรับครู
- [ ] **Real-time Missing List**: แสดงรายชื่อนักเรียนที่ "ยังไม่ได้เช็คชื่อ" ในหน้า Dashboard ของครูแบบ Real-time
- [ ] **Manual Overwrite**: ครูสามารถกดเปลี่ยนสถานะจาก "ขาด" เป็น "มา" หรือ "สาย" ได้เองในกรณีที่นักเรียนมาถึงห้องแล้วแต่ลืมเช็คชื่อ หรือโทรศัพท์มีปัญหา

### สถิติและการแจ้งเตือนขั้นสูง
- [ ] **Absence Threshold Warning**: หากนักเรียนขาดเรียนวิชาเดิมซ้ำกันเกิน 3 ครั้ง ระบบจะส่งการแจ้งเตือนระดับ "สำคัญมาก" ไปยังครูที่ปรึกษาและผู้ปกครอง
- [ ] **Late vs Absent**: กำหนดเกณฑ์เวลา เช่น 
  - 0-15 นาที = **มาเรียน (Present)**
  - 15-30 นาที = **สาย (Late)**
  - เกิน 30 นาที = **ขาดเรียน (Absent)** อัตโนมัติ

## 🛠️ 9. การปรับปรุงระบบ Component และ Code Clean (Component Refactoring)
ไอเดียการย้าย Logic ที่ใช้ซ้ำๆ ไปเป็น Global Function ใน `src/lib/ui.js` เพื่อให้โค้ดสะอาดและดูแลง่ายขึ้น:

- [ ] **Confirmation Modal with Countdown**:
  - สร้างฟังก์ชัน `showConfirmModal({ title, desc, action, onConfirm })` 
  - ย้าย Logic การนับถอยหลังและการจัดการวงกลม (Circular Countdown) เข้าไปอยู่ในฟังก์ชันเดียว
  - ใช้แทนการเขียน Modal ซ้ำๆ ในหน้า Approve และ Activity
- [ ] **Reject Detail Modal**:
  - สร้างฟังก์ชัน `showRejectDetail(data)` เพื่อแสดงเหตุผลการปฏิเสธและข้อมูลผู้ตรวจสอบ
  - ใช้แทนการสร้าง Modal แบบ Static ใน HTML ทุกหน้า
- [ ] **Profile Header Component**:
  - สร้างฟังก์ชัน `renderProfileHeader(userData, avatarUrl)` 
  - ใช้เรนเดอร์ส่วนบนของ Dashboard ทุกหน้า (Student, Teacher, Leader, Admin) เพื่อให้ดีไซน์เป็นมาตรฐานเดียวกัน
- [ ] **Loading Overlay / Spinner**:
  - สร้างฟังก์ชัน `showLoading()` และ `hideLoading()` สำหรับแสดงสถานะการดึงข้อมูลจาก Supabase
  - ช่วยให้ User Experience (UX) ดูลื่นไหลและเป็นมืออาชีพ
- [ ] **Circular Countdown Utility**:
  - แยก Logic การวาดวงกลมจับเวลา (`stroke-dashoffset`) ออกมาเป็น Utility function 
  - เพื่อให้หน้าอื่นๆ (เช่น "Verify All") เรียกใช้ได้ทันทีโดยไม่ต้องคำนวณใหม่
- [ ] **Standardized Back Button**:
  - สร้าง Component สำหรับปุ่มย้อนกลับสไตล์ Neubrutalism ที่ใช้บ่อยในหน้าย่อยต่างๆ
