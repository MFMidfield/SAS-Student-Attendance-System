# 🚀 Lunar PJ - Features List (รายการฟีเจอร์ปัจจุบัน)

ระบบจัดการและบันทึกเวลาเรียนสไตล์ **Neubrutalism** ผสมผสานระบบตรวจสอบสิทธิ์และตารางเวลาเรียนที่แม่นยำ ปัจจุบันระบบประกอบด้วยฟีเจอร์หลัก ๆ ดังนี้:

---

## 👨‍🎓 1. ระบบเช็คชื่อและยื่นใบลาสำหรับนักเรียน (Student Attendance & Leave)
*   **ระบบเช็คชื่อเข้าเรียนแบบ Real-time (Smart Present):**
    *   ระบบตรวจสอบวันเวลาปัจจุบันเทียบตารางเรียน (`schedule`) ประจำห้องอัตโนมัติ เพื่อดึงคาบเรียนและชื่อวิชามาล็อกไว้ตอนสแกนเช็คชื่อ
    *   นักเรียนไม่ต้องเลือกวิชาเอง ป้องกันการสับสนหรือเลือกวิชาผิด
*   **การควบคุมเวลาสแกนเข้มงวด (No Class Check-in Protection):**
    *   หากนอกเหนือเวลาเรียนตามตาราง ปุ่มยืนยันเข้าเรียนจะถูกล็อกและแสดงข้อความ `"No Class Now"` ป้องกันนักเรียนกดเช็คชื่อย้อนหลังนอกคาบเรียน
*   **ระบบยื่นใบลาแบบมีเงื่อนไข (Conditional Leave System):**
    *   เลือกประเภทการลาได้ 3 แบบ: ลาป่วย (Sick), ลากิจ (Personal), หรือทำกิจกรรม (Activity)
    *   ระบุช่วงเวลาได้ละเอียด (Leave Scope): ลาทั้งวัน (Full Day), ครึ่งวันเช้า (Morning), หรือครึ่งวันบ่าย (Afternoon)
    *   **บังคับกรอกเหตุผล (Mandatory Note):** สำหรับการลา กิจ/ป่วย/กิจกรรม ต้องระบุเหตุผลประกอบเสมอ ห้ามเว้นว่าง
*   **ระบบป้องกันความขัดแย้งของข้อมูล (Data Conflict Prevention):**
    *   บล็อกการส่งใบลาซ้อน หากวันนั้นมีการยื่นใบลาแบบทั้งวัน (Full Day) อยู่แล้ว
    *   บล็อกการยื่นใบลาแบบทั้งวัน หากวันนั้นมีประวัติเช็คชื่อเข้าเรียน (Present) แล้ว
*   **ระบบยืนยันแบบหน่วงเวลา (Countdown Confirmation):**
    *   เมื่อกดส่งข้อมูลเช็คชื่อ จะมีระบบนับถอยหลัง 5 วินาทีก่อนปุ่มยืนยันจริงจะเปิดให้กด เพื่อส่งเสริมความรอบคอบและป้องกันความผิดพลาด (Human Error)

---

## 📅 2. ระบบจัดการตารางสอน (Schedule Management - Admin/Teacher)
*   **การเพิ่ม/แก้ไขวิชาสอน (Add & Edit Schedule):**
    *   ผู้ที่มีสิทธิ์ (Admin และ Teacher) สามารถเพิ่มข้อมูลตารางสอนรายคาบได้
*   **การล็อกคาบเรียนกับเวลามาตรฐาน (Period-Time Locking):**
    *   เมื่อเลือกคาบเรียน (คาบ 0 ถึง คาบ 10) ระบบจะสวมค่าและล็อกเวลาเรียนให้อัตโนมัติ (เช่น คาบ 1 ➔ 08:30 - 09:20, คาบ 0 ➔ 08:00 - 08:30)
    *   ตัวเลือกแสดงผลเวลาเปลี่ยนเป็น `div` แสดงเวลาปกติแทนช่องกรอกอินพุตแบบเดิม เพื่อความชัดเจนและง่ายต่อสายตา
*   **ข้อยกเว้นพิเศษสำหรับ คาบ 0 (Homeroom Bypass Rules):**
    *   ข้ามขั้นตอนการเช็ควิชาว่างสำหรับคาบ 0 เพื่อให้กดบันทึกได้ทันที
    *   ระบบใส่ค่าชื่อวิชาเป็น `"Homeroom"` ให้อัตโนมัติเมื่อกดบันทึกในกรณีที่เว้นช่องวิชาว่างไว้
*   **ระบบควบคุมข้อมูลพิเศษ (Special Subject Lock):**
    *   คาบเรียนพิเศษ/โฮมรูม (Period 0) หรือคาบที่ระบุวิชาเป็น Homeroom จะถูกล็อกห้ามลบและห้ามแก้ไขฟิลด์สำคัญเพื่อความปลอดภัยของระบบหลัก
*   **การค้นหาและฟิลเตอร์ขั้นสูง (Search & numerical Classroom Filters):**
    *   ช่องค้นหาตารางเรียนแบบพิมพ์แล้วหาเจอทันที (Instant Search) ค้นได้จากชื่อวิชา, ชื่อครู, ห้องเรียน หรือคาบเรียน
    *   กล่องเลือกตัวกรองห้องเรียน (Classroom Filter) จะจัดเรียงห้องจากน้อยไปหามากตามหลักตัวเลข (เช่น 4/9 ➔ 5/9 ➔ 6/9) โดยมีกลไกจัดเตรียมห้องต่ำสุดเป็นค่าเริ่มต้น (Default Select) ทันทีที่เปิดหน้าจอ
    *   **หมายเหตุ:** นำตัวเลือก "ALL ROOMS" ในหน้าจอตั้งค่าตารางเรียนออก เพื่อไม่ให้เกิดความสับสนและให้โฟกัสที่การจัดการตารางทีละห้องเรียน
*   **การลบวิชาแบบป้องกันการกดพลาด (Delete Timer Modal):**
    *   ปุ่มลบตารางเรียนในโหมดแก้ไขจะมีหน้าต่างแจ้งเตือนนับถอยหลัง 5 วินาที ก่อนปลดล็อกให้กดยืนยันลบจริง เพื่อเลี่ยงอุบัติเหตุการลบข้อมูลผิดพลาด

## 👨‍🏫 3. ระบบอนุมัติและตรวจสอบสถานะเข้าเรียน (Attendance Approval - Teacher/Leader)
*   **ตารางตรวจสอบสถานะประจำวัน (Daily Logs Inspector):**
    *   หน้าจอสรุปข้อมูลการสแกนเรียน/ใบลาของนักเรียนในห้องเรียนประจำชั้นแยกตามวันที่ต้องการ
    *   ค้นหาตามชื่อนักเรียน หรือรหัสประจำตัวได้แบบเรียลไทม์ (Instant Search)
*   **การพิจารณาอนุมัติใบลา/การมาเรียน (Verification System):**
    *   อาจารย์หรือหัวหน้าห้องสามารถกดเข้ามาตรวจสอบใบสแกน/ใบลาที่สถานะเป็น "รอดำเนินการ (Pending)"
    *   แสดงรายละเอียดข้อมูลใบลาประกอบการตัดสินใจ (ประเภทการลา, ช่วงเวลาที่ลา, เหตุผลแนบ)
    *   **การไม่อนุมัติ (Reject Logic):** หากเลือกปฏิเสธคำขอ (Rejected) ระบบจะบังคับให้อาจารย์ผู้ตรวจบันทึกเหตุผลกำกับ (Teacher Note)
    *   **ปุ่มหน่วงเวลา 3 วินาที:** การกดอนุมัติ/ไม่อนุมัติ จะมีระบบนับถอยหลัง 3 วินาทีก่อน เพื่อให้ตรวจสอบข้อมูลบนจอให้แน่ใจก่อนกดจริง
*   **หน้าต่างแสดงสาเหตุที่ถูกปฏิเสธ (Reject Detail view):**
    *   นักเรียนหรือครูสามารถกดดูรายละเอียดเหตุผลที่ใบสแกนหรือใบลาถูกปฏิเสธได้ โดยระบบจะระบุข้อมูลเหตุผล (Teacher Note), ชื่อผู้ลงมติปฏิเสธ, บทบาทของผู้ปฏิเสธ และวันเวลาที่ทำรายการอย่างครบถ้วน
*   **การอนุมัติแบบกลุ่ม (Verify All Pending):**
    *   อาจารย์สามารถกดเลือกตรวจสอบรายชื่อนักเรียนที่มีคำค้างพิจารณาทั้งหมดในหน้าจอเดียว
    *   สามารถเปลี่ยนสถานะนักเรียนแต่ละคน (มาเรียน, ลาป่วย, ลากิจ, ขาดเรียน) และกดยืนยันคำขออนุมัติพร้อมกันทั้งหมดได้ทันทีในครั้งเดียว (พร้อมระบบนับถอยหลัง 5 วินาทีตรวจสอบซ้ำ)

---

## 📊 4. ระบบสรุปและส่งออกข้อมูล (Data & Logs Export - Admin)
*   **ระบบตรวจสอบบันทึกย้อนหลัง (Admin Logs Explorer):**
    *   หน้ารวม Log ประวัติการเข้าเรียนทั้งหมดของนักเรียน
    *   กรองข้อมูลตามช่วงวันที่ (Date Range), ค้นหารายวิชา หรือพิมพ์หาข้อมูลรายบุคคลได้
*   **ดาวน์โหลดรายงานเป็นไฟล์ Excel (Export to Excel):**
    *   มีปุ่มกดดาวน์โหลดตาราง Log การเข้าเรียนที่เลือกออกมาเป็นไฟล์ `.xlsx` (โดยใช้ SheetJS Library) เพื่อนำไปเปิดแก้ไขต่อหรือพิมพ์รายงานประจำสัปดาห์/เดือน

---

## 🔐 5. ระบบความปลอดภัย สิทธิ์การใช้งาน และสื่อ (User Auth, RBAC & Assets Management)
*   **การสมัครและล็อกอินใช้งาน (Auth Flow):**
    *   ล็อกอินเข้าใช้งานระบบด้วยระบบรักษาความปลอดภัยของ Supabase Auth
    *   มีฟังก์ชัน Toggle Show/Hide Password หน้าล็อกอินและสมัครสมาชิกเพื่อป้องกันการพิมพ์ผิดพลาด
*   **สิทธิ์การเข้าใช้งานแยกตามบทบาท (Role-Based Access Control - RBAC):**
    *   **Admin:** ดูแลระบบโดยภาพรวม, อนุมัติสิทธิ์ (Role) สมาชิก, แก้ไขข้อมูลประวัตินักเรียน, จัดการตารางเรียน และดู Logs ทั้งหมด
    *   **Teacher:** จัดการตารางสอน, ตรวจสอบและอนุมัติเวลาเรียน/ใบลาของนักเรียนในห้องประจำชั้นของตนเอง
    *   **Leader:** เข้าถึงข้อมูลของเพื่อนในห้องเรียนและช่วยอนุมัติการเช็คชื่อตามห้องเรียนที่ได้รับมอบหมาย
    *   **Student:** ดูตารางเรียนประจำวัน, เช็คชื่อมาเรียน (ตามช่วงเวลาเรียนจริง), ยื่นเรื่องใบลาเมื่อเกิดเหตุจำเป็น และดูสถิติใบลาของตนเอง
*   **การรักษาความปลอดภัยของฐานข้อมูล (Database Security):**
    *   เปิดใช้งาน Row Level Security (RLS) บน Supabase ป้องกันไม่ให้นักเรียนแอบข้ามหน้าจอไปแก้ไขตารางเรียนหรือสิทธิ์ผู้อื่น
    *   มี Database Trigger Functions ป้องกันไม่ให้หน้าบ้าน (Frontend) แก้ไข Role ของตนเองเพื่อเลื่อนสิทธิ์โดยตรง (ต้องให้ Admin เปลี่ยนให้ในตารางโปรไฟล์เท่านั้น)
*   **การจัดการรูปภาพสื่อในห้องเรียน (Student/Classroom Media Management):**
    *   มีโมดูลสำหรับการอัปโหลดและจัดการภาพแบนเนอร์ห้องเรียน (`student-img-banner`) และโลโก้ประจำตัว (`student-img-logo`) 
    *   เก็บและตรวจสอบพิกัดรูปแยกตามรายบัญชีผ่านตาราง `user_assets` บนฐานข้อมูล
